# AcadeGrade: Student (Authenticated) Features

This document outlines the core features available to authenticated students on the AcadeGrade platform. It is based on a full code audit of the `app/(student)` directory.

## 1. Main Dashboard (`/dashboard`)
- **Overview Arc**: A dynamic SVG radial progress arc (`CGPAArc`) displaying the current CGPA out of 5.0, accompanied by a visual degree class badge (e.g., "Second Class Upper").
- **Metric Toggle**: Users can seamlessly toggle the dashboard to display the official **CGPA** or the **Performance Index (PI)** as their primary metric. This preference is saved directly to Firestore.
- **KPI Cards**: Tracks Total Credits (CU), Current Semester GPA, Total Courses Completed, and an **"At Risk"** counter (highlighted in red) tracking courses with an 'E' or 'F' grade.
- **Trend Chart**: A Recharts line graph comparing official CGPA against True Mastery (PI) across all recorded semesters.
- **AI Summary Card**: Displays the latest cached "Degree Outlook" generated by the AI. It highlights if the data is "Stale" and includes a 12-hour cooldown refresh button to prevent API abuse.
- **Web Share API**: A native "Share Progress" button that uses the device's native sharing sheet (if available) or falls back to clipboard copying.

## 2. Results & Course Management (`/results` & `/results/[semesterId]`)
- **Semester Creation (`/results/new`)**: Users create semesters by selecting their Academic Level (100L - 1000L), Semester (1st/2nd), and Academic Session (e.g., "2025/2026").
- **Smart Data Entry**:
  - **Manual Entry**: Adding Course Code, Units, and Grade/Score. Autocomplete pulls from the Admin's Global Course Catalog.
  - **AI Extraction (OCR)**: Via `/api/results/extract`. Users can upload an image or PDF of a physical result slip. The backend converts the file to a Base64 string and uses **Gemini 3.1 Flash-Lite Multimodal** to parse it into an array of courses automatically.
- **Real-time Recalculation**: The platform instantly recalculates the semester GPA, PI, and total degree CGPA whenever a course is added, edited, or deleted.

## 3. Automated Transcripts (`/transcript`)
- **PDF Export (`/api/transcript/generate`)**: High-quality, dynamically generated PDF transcripts using `jspdf` and `jspdf-autotable`. The backend actively handles the PDF generation.
- **Photo Embedding**: The user can toggle a "Show Photo" switch. If active, the system fetches the user's avatar from Firebase Storage, converts it to a Base64 string to bypass CORS issues, and embeds it directly into the PDF header.
- **Public Sharing (`/api/transcript/share`)**: Creates a secure, shareable URL snapshot of the transcript. The generated `shareId` points to a frozen record in Firestore, allowing third parties to view the transcript without logging in.

## 4. AI Academic Insights (`/insights`)
- **Rate Limiting**: The frontend enforces a strict 12-hour cooldown timer (`rateLimitCooldown`) between full AI regenerations, displaying a live countdown clock to the user to manage API costs.
- **Forecast Tab**: Uses `simple-statistics` (linear regression) on the student's historical Performance Index (PI) to mathematically project their CGPA two semesters into the future, rendering a dashed Recharts projection line.
- **What-If Forecaster**: An interactive calculator where the student slides a target CGPA. It instantly computes the exact GPA needed in remaining semesters to mathematically achieve the target.
- **Risk Analysis**: Highlights failing or declining courses and computes an overall risk gauge based on the slope of the PI trend.
- **Written Analysis**: A 400-600 word personalized summary generated by the AI (DeepSeek/Gemini), complete with a character-by-character typing animation upon initial load.

## 5. Notifications & Real-time Sync (`/notifications`)
- **Multi-Channel Delivery**: The system supports both **In-App Push Notifications** (via FCM) and **Email Notifications** (via Nodemailer/Gmail SMTP) depending on the event type (e.g., Admin Broadcasts vs. OTPs).
- **In-App Bell**: Tracks unread announcements and system alerts, syncing instantly via Firebase Realtime Database (RTDB) for minimal latency.
- **FCM Token Array Architecture**: To support users logging in across multiple devices (e.g., Phone and Laptop simultaneously), the Firestore user document stores an `fcmTokens` array (`string[]`). When a user logs in and grants permission, their specific device token is pushed into this array. When the backend sends a notification, it loops through the entire array to notify all active devices.
- **Active Token Cleanup (Logout)**: When a user clicks "Logout", the client explicitly fires a request to `/api/user/fcm-token` (DELETE) with their current device's token. The backend uses `adminDb.collection('users').doc(uid).update({ fcmTokens: FieldValue.arrayRemove(token) })` to slice that specific token out of the array before actually logging them out of Firebase Auth. This ensures they stop receiving pushes on that device instantly.
- **Passive Token Cleanup**: If a push notification fails to deliver because a token has naturally expired or the user revoked browser permissions, the backend catches the `messaging/invalid-registration-token` error and aggressively prunes the dead token from the database array, keeping the database extremely clean.

## 6. Profile & Settings (`/settings`)
- **Avatar Uploads**: Direct integration with Cloudinary using an unsigned upload preset (`acadegrade_avatars`), instantly saving the `secure_url` back to Firestore.
- **Platform Preferences**: Toggles for the default grading metric (CGPA vs PI) and the Record Mode ("From Scratch" vs "Complete Record").
- **Notification Toggles**: Granular control over which events trigger emails or pushes (e.g., "Semester Saved", "AI Insights", "Admin Broadcasts").
- **Security Actions**: In-app password updates and a destructive "Delete Account" flow that requires re-authentication before wiping all student data from Firebase.
